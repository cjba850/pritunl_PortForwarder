Add Port Forward Rule
Port ranges (e.g. 8000-8010) are supported on either side of an inbound rule. A range on one side and a single port on the other is valid (many-to-one or one-to-many); two differently-sized ranges are not.
Outbound rules pin the source port of new outbound connections from this endpoint via SNAT/MASQUERADE (e.g. so a remote service that expects all traffic from one fixed port sees it consistently) — they're inserted ahead of Pritunl's own default MASQUERADE rule so they actually take effect. They can't redirect reply traffic that's already part of an established connection (the kernel reuses cached NAT state for that) — only genuinely new outbound connections.
